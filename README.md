# Quizzboodle
Multiplayer quiz game using Streamlit and Supabase

### How to create a quiz (JSON) file (example: quiz_data.json)

You can use the included **Quiz Creator** desktop app (`quiz_creator.py`) to build a quiz with a GUI, or write the JSON file by hand. The file must follow this structure:

```json
{
  "display_time": 5,
  "wait_before_next": 3,
  "questions": [
    {
      "question": "What is the capital of France?",
      "question_image": "",
      "answers": [
        { "text": "Paris",  "image": "" },
        { "text": "Berlin", "image": "" },
        { "text": "Madrid", "image": "" },
        { "text": "Rome",   "image": "" }
      ],
      "correct_answer_index": 0
    }
  ]
}
```

Key fields:
- **`display_time`** — seconds players have to answer each question
- **`wait_before_next`** — seconds to pause after the timer expires before moving to the next question
- **`correct_answer_index`** — zero-based index of the correct answer in the `answers` array
- **`question_image`** / **`image`** — optional path to an image; leave as `""` if not used. (The path to an image must start with the folder it is in!)

---

### How to upload a quiz and start it as admin

1. Open the app in a browser and enter `admin` as the player name, then click **Join**.
2. You are taken to the **Admin Room**. The admin is not visible to players.
3. Under **Upload Quiz File**, click **Browse files** and select your `.json` quiz file. A success message confirms how many questions were loaded.
4. Wait for players to join — the lobby list updates automatically every 3 seconds.
5. When all players are in the lobby, click **▶ Start Quiz for Everyone**. All players are sent to the countdown page simultaneously.
6. When the game ends and all players have reached the results screen, click **🔴 End Game & Reset Everything** to clear the lobby and scores and return the game to a waiting state ready for the next round.

---

### How to play the game

1. Open the app URL in a browser and enter your name, then click **Join**.
2. You are taken to the **Lobby**. Wait here until the host starts the quiz. The page updates automatically.
3. When the host starts, a short **countdown** appears, then the first question is shown.
4. Each question has a timer (set by the quiz author). Click one of the answer buttons before time runs out — you can change your selection until the timer expires.
5. After all questions are answered, you are taken to the **Results** screen showing your score and the leaderboard.
6. To play again, wait for the host to reset the game and re-enter your name on the login screen.



## Supabase Setup

If you want to run this project with your own Supabase account, follow these steps.

### 1. Create a Supabase project

1. Go to [https://supabase.com](https://supabase.com) and sign in and create a free account.
2. Click **New project**, give it a name (e.g. `quizzboodle`), choose a region, and set a database password.
3. Wait for the project to finish provisioning.

---

### 2. Create the required tables

Open the **SQL Editor** in the Supabase dashboard and run the following SQL to create all four tables:

```sql
-- Players waiting to join a game
CREATE TABLE lobby (
    id   bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    player_name text NOT NULL UNIQUE
);

-- Single row that tracks whether a game is running
CREATE TABLE game_state (
    id    bigint PRIMARY KEY,
    state text   NOT NULL
);

-- Seed the required row (must exist before the app starts)
INSERT INTO game_state (id, state) VALUES (1, 'WAITING');

-- Per-player scores for each game
CREATE TABLE scores (
    id          bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    player_name text,
    score       integer,
    total       integer
);

-- Stores the uploaded quiz JSON (always a single row)
CREATE TABLE quiz_config (
    id   bigint PRIMARY KEY,
    data jsonb
);
```

---

### 3. Disable Row Level Security (RLS)

By default Supabase enables RLS on new tables, which blocks all reads and writes from the app. Either disable it or add permissive policies for each table.

**Quickest option — disable RLS on all four tables:**

```sql
ALTER TABLE lobby       DISABLE ROW LEVEL SECURITY;
ALTER TABLE game_state  DISABLE ROW LEVEL SECURITY;
ALTER TABLE scores      DISABLE ROW LEVEL SECURITY;
ALTER TABLE quiz_config DISABLE ROW LEVEL SECURITY;
```

---

### 4. Get your project credentials

In the Supabase dashboard go to **Project Settings → API**.

Copy:
- **Project URL** (looks like `https://xxxxxxxxxxxx.supabase.co`)
- **anon / public key** (the long `eyJ...` string under *Project API keys*)

---

### 5. Create the Streamlit secrets file

Create the file `.streamlit/secrets.toml` in the project root (this file is gitignored and must never be committed):

```toml
[supabase]
url = "https://xxxxxxxxxxxx.supabase.co"
key = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

Replace the placeholders with your actual URL and anon key from step 4.

---

### 6. Install dependencies and run

```bash
pip install -r requirements.txt
streamlit run app.py
```

The admin page is accessed by entering `admin` as the player name on the login screen.
